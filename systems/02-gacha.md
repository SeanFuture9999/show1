# 抽卡系統規格

## 基本資訊
- **優先級**：Must
- **依賴系統**：囂張值系統、章節管理
- **被依賴**：忍/用系統、廣告系統

---

## 1. 設計目標

1. 製造驚喜感和期待感
2. 隨機性讓每次遊玩體驗不同
3. 廣告變現入口（額外抽卡機會）

### 非目標
- 不做複雜的保底/天井系統（MVP 階段）
- 不做卡牌升級/合成

---

## 2. 核心機制

### 抽卡觸發條件
- 劇情推進中，壞人事件累積到一定程度 → 懷錶自動發光
- 每章固定 3-5 次抽卡機會（劇情節點綁定）
- 看廣告可獲得額外 1 次抽卡

### 抽卡流程
```
懷錶發光提示
  ↓
玩家點擊懷錶
  ↓
懷錶打開動畫（轉盤/光效）
  ↓
卡牌飛出 → 翻轉揭示
  ↓
顯示卡牌資訊（名稱、效果、稀有度）
  ↓
進入忍/用選擇
```

### 卡池機率
| 稀有度 | 機率 | 顏色 | 效果範圍 |
|--------|------|------|----------|
| N 普通 | 50% | 灰白 | 囂張值 -8% |
| R 稀有 | 30% | 藍色 | 囂張值 -15% |
| SR 超稀有 | 15% | 紫色 | 囂張值 -25% |
| SSR 傳說 | 5% | 金色 | 囂張值 -40% |

### 卡牌類型（依打臉方式分）
| 類型 | 範例卡牌 | 打臉方式 |
|------|----------|----------|
| 💼 商業 | 財務報告、合約漏洞、股權證明 | 揭露對方商業弱點 |
| 🤝 人脈 | 董事長名片、記者朋友、律師電話 | 搬出更大的靠山 |
| 📱 情報 | 監控錄影、聊天記錄、銀行流水 | 揭露對方秘密 |
| 💡 智慧 | 談判技巧、心理戰術、商業知識 | 智商碾壓 |
| 🎭 演技 | 裝窮、裝傻、苦肉計 | 反向操作讓壞人露出破綻 |

### MVP 卡牌數量
| 稀有度 | 數量 | 合計 |
|--------|------|------|
| N | 10 張 | |
| R | 8 張 | |
| SR | 5 張 | |
| SSR | 3 張 | |
| **合計** | **26 張** | |

## 3. 資料結構

```json
Card {
  id: string,
  name: string,           // "財務報告"
  rarity: "N"|"R"|"SR"|"SSR",
  type: "business"|"connection"|"intel"|"wisdom"|"acting",
  base_damage: float,     // 基礎囂張值減少量
  description: string,    // 卡牌描述
  slap_text: string,      // 打臉時的台詞
  art_id: string          // 對應像素插畫 ID
}

GachaPool {
  chapter_id: string,
  available_cards: Card[],  // 本章可抽到的卡
  rates: { N: 0.5, R: 0.3, SR: 0.15, SSR: 0.05 }
}
```

## 4. UI/UX 需求

### 抽卡演出（依稀有度）
| 稀有度 | 演出時長 | 特效 |
|--------|----------|------|
| N | 1 秒 | 懷錶輕閃，卡牌滑出 |
| R | 1.5 秒 | 藍光，卡牌翻轉 |
| SR | 2.5 秒 | 紫光爆發，懷錶震動 |
| SSR | 4 秒 | 全屏金光，懷錶裂開再合上，卡牌從光柱降下 |

## 5. 迭代計畫

### MVP
- 單卡池，固定機率
- 26 張基礎卡牌
- 簡單翻轉動畫

### V1.1
- 章節限定卡（特定章節才有的卡）
- 10 連抽保底 SR

### 未來
